# 상세 리포트 설계서 v5.0

> 온보딩 직후 생성되는 AI 상세 리포트의 전체 설계.
> Claude Sonnet 4.6 배치 API (300K 출력) 기반, 섹션별 JSON 구조화.

---

## 1. 핵심 원칙

### 1-1. 데이터 분리 원칙

```
system_config (운영팀 관리)  →  통계, 평균, 비교 데이터 (숫자)
user profile (온보딩 입력)   →  개인 재무 데이터 (숫자)
AI (Claude Sonnet 4.6)      →  분석 서술 + 인사이트 (텍스트만)
```

- **AI는 숫자를 만들지 않는다.** 모든 수치는 system_config 또는 user profile에서 가져온다.
- AI가 받는 프롬프트에 계산 완료된 숫자를 직접 삽입하고, AI는 해석/서술만 한다.
- 할루시네이션 원천 차단.

### 1-2. 교육 앱 원칙

- **추천하지 않는다.** "이런 게 있어, 이런 특징이야, 관심 있으면 알아봐" 톤.
- 특정 금융 상품, 종목, 증권사를 지목하지 않는다.
- 확정적 투자 권유 금지. 면책 문구 자동 삽입.

### 1-3. MVP 데이터 제약

온보딩에서 받는 데이터가 전부:

| 필드 | 타입 | 예시 |
|---|---|---|
| nickname | string | "시은" |
| age | int | 28 |
| retirementAge | int | 55 |
| pensionStartAge | int | 65 |
| monthlyIncome | int (원) | 2,500,000 |
| monthlyFixedCost | int (원) | 1,000,000 |
| monthlyVariableCost | int (원) | 800,000 |

서버에서 계산되는 값:

| 필드 | 계산식 | 예시 |
|---|---|---|
| monthlyExpense | fixedCost + variableCost | 1,800,000 |
| surplus | income - expense | 700,000 |
| expenseRatio | expense / income | 0.72 |
| grade | expenseRatio 기준 RED/YELLOW/GREEN | RED |
| variableCost.daily | (income - fixedCost) / 이번달일수 | 48,000 |
| investmentPeriod | retirementAge - age | 27년 |
| vestingPeriod | pensionStartAge - retirementAge | 10년 |

**못 받는 것:** 카드 내역, 계좌 잔액, 지출 카테고리 상세, 직업 유형, 기존 자산/부채, 가족 구성

## 2. 리포트 섹션 구조 (12개)

### 전체 스토리라인

```
나는 누구인가 → 내 위치는 어디인가 → 이대로면 어떻게 되는가 → 어떻게 바꿀 수 있는가
```

| Part | # | 섹션 | 핵심 가치 |
|---|---|---|---|
| **Part 1. 내 돈의 현재** | 1 | 재무 건강 진단 | 한눈에 내 상태 파악 |
| | 2 | 돈의 흐름 | 돈이 어디로 빠지는지 |
| **Part 2. 나의 위치** | 3 | 또래 비교 | 동기부여/위기감 |
| | 4 | 세계 속 한국 | 교육/맥락 이해 |
| **Part 3. 미래 시뮬레이션** | 5 | 자산 성장 시뮬레이션 | 복리 효과 체감 |
| | 6 | 은퇴 시뮬레이션 | 위기감/장기 계획 |
| | 7 | 생애 이벤트 대비 | 현실적 목표 설정 |
| **Part 4. 전략** | 8 | 등급별 로드맵 | 즉시 실행 가능한 방향 |
| | 9 | 잉여자금 늘리기 시뮬레이션 | "이만큼 바뀔 수 있어" |
| | 10 | 금융 교육 | 등급별 맞춤 지식 |
| **Part 5. 부록** | 11 | 금융 용어 사전 | 두고두고 참고 |
| | 12 | 12개월 재무 캘린더 | 연간 재무 관리 |

## 3. 섹션별 상세 스펙

### Section 1. 재무 건강 진단

**목적:** 유저의 전체 재무 상태를 점수화하여 한눈에 보여주기

**입력 데이터:**
- user: monthlyIncome, monthlyFixedCost, monthlyVariableCost, surplus, expenseRatio, grade, age, retirementAge
- system_config: peer_savings_rate_{나이대}, peer_avg_expense_ratio_{나이대}

**서버에서 계산 (AI에게 주기 전):**

| 축 | 계산식 | 만점 기준 |
|---|---|---|
| 지출 비율 점수 | (1 - expenseRatio) × 100 점으로 환산 | 지출비율 30% 이하 = 만점 |
| 잉여자금 비율 | (surplus / income) × 100 | 30% 이상 = 만점 |
| 고정비 비중 | 고정비 / 총지출 → 낮을수록 유연 | 50% 이하 = 만점 |
| 변동비 여유도 | 유저 변동비 vs 또래 평균 변동비 | 평균 이하 = 만점 |
| 투자 가용 기간 | 은퇴까지 남은 년수 환산 | 25년 이상 = 만점 |

**JSON 출력 구조:**

### Section 5. 자산 성장 시뮬레이션

**목적:** 현재 잉여자금을 다양한 방법으로 굴렸을 때 미래 자산 시각화

**입력 데이터:**
- user: surplus (월 잉여자금)
- system_config: savings_interest_rate, avg_stock_return_kr, avg_bond_return

**서버에서 계산 (복리 공식):**

```
FV = PMT × ((1 + r/12)^(n×12) - 1) / (r/12)

PMT = 월 잉여자금
r = 연 수익률
n = 년수
```

3가지 시나리오 × 5개 시점 = 15개 숫자 → 전부 서버에서 계산

**JSON 출력 구조:**


### Section 6. 은퇴 시뮬레이션

**목적:** 은퇴~연금수령 공백기를 시각화하고, 필요 자금 계산

**입력 데이터:**
- user: age, retirementAge, pensionStartAge, surplus
- system_config: national_pension_avg_monthly, min_living_cost_retirement, comfortable_living_cost_retirement

**서버에서 계산:**

```
investmentPeriod = retirementAge - age          (자산 축적 기간)
vestingPeriod = pensionStartAge - retirementAge (연금 공백기)
gapFundMin = vestingPeriod × 12 × min_living_cost_retirement
gapFundComfort = vestingPeriod × 12 × comfortable_living_cost_retirement
monthlyGapAfterPension = comfortable_living_cost - national_pension_avg
```

### Section 7. 생애 이벤트 대비

**목적:** 주요 생애 이벤트의 평균 비용과 현재 저축 속도 기준 달성 시점

**입력 데이터:**
- user: age, surplus
- system_config: avg_wedding_cost, avg_jeonse_seoul, avg_child_cost_0_18, avg_car_5yr_cost

**서버에서 계산:**

```
각 이벤트 달성까지 필요 개월 = 이벤트 비용 / 월 잉여자금
달성 시 나이 = 현재 나이 + (필요 개월 / 12)
```

**JSON 출력 구조:**

```json
{
  "section": "life_events",
  "monthlySurplus": 700000,
  "events": [
    {
      "name": "결혼",
      "avgCost": 35000000,
      "monthsNeeded": 50,
      "achieveAge": 32,
      "icon": "wedding"
    },
    {
```

## 5. 생성 파이프라인

### 아키텍처

```
[유저 온보딩 완료]
      │
      ▼
[서버: 데이터 수집]
  ├── finance_profiles → 유저 재무 데이터
  ├── system_config → 전체 configMap
  └── 서버에서 모든 숫자 계산 (시뮬레이션, 등급, 비교 등)
      │
      ▼
[서버: 프롬프트 조립]
  ├── system prompt: AI 역할, 규칙, 금지사항
  ├── user prompt: 계산된 숫자 + 섹션별 지시
  └── 출력: JSON 12개 섹션
      │
      ▼
[Claude Sonnet 4.6 배치 API]
  ├── beta header: output-300k-2026-03-24
  ├── max_tokens: 200000 (여유 있게)
  └── 1회 호출로 12개 섹션 전체 생성
      │
      ▼
[서버: 후처리]
  ├── JSON 파싱 + 검증
  ├── 금지 표현 필터링
  ├── 면책 문구 삽입
  └── DB 저장 (detailed_reports)
      │
      ▼
[프론트: 렌더링]
  ├── 섹션별 카드/탭 UI
  ├── 차트 라이브러리로 인터랙티브 시각화
  └── PDF 다운로드 (선택)
```

### API 호출 방식

```typescript
// 배치 API 사용 (300K 출력, 50% 할인)
const response = await client.messages.create({
  model: 'claude-sonnet-4-6-20260514',
  max_tokens: 200000,
  // beta header로 300K 출력 활성화
  betas: ['output-300k-2026-03-24'],
  system: SYSTEM_PROMPT,
  messages: [{ role: 'user', content: buildPrompt(profile, configMap, calculations) }],
});
```

### 프롬프트 전략

```
[system prompt]
- 너는 머니런 AI 재무 분석가
- 교육 앱 톤: 추천이 아닌 교육
- 반말, 친근하지만 전문적
- 내가 주는 숫자만 사용. 새로운 숫자 만들지 마
- 금지: 특정 상품 추천, 확정 투자 권유, 수익 보장 표현
- 응답: JSON only, 12개 섹션 배열

[user prompt]
- 유저 데이터 (전부 숫자로)
- system_config 데이터 (전부 숫자로)
- 서버에서 미리 계산한 값들 (시뮬레이션, 비교, 등급 등)
- 각 섹션별 ai_narrative만 작성하라는 지시
```

핵심: **숫자/차트 데이터는 서버에서 계산해서 JSON에 직접 넣고, AI에게는 ai_narrative 텍스트만 생성하게 한다.**
