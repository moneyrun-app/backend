# 퀴즈 시스템 공공데이터 연동 기획

> 2026.04.06 기획. 공공 API 데이터 수집 → AI 가공 → 퀴즈 대량 생성.
> 다음 세션에서 이어서 개발할 때 이 문서 참고.

---

## 배경

- 현재 퀴즈 30개 수동 삽입 → 하루 10개 배정이면 3일에 고갈
- 목표: 공공데이터 기반 ~4,000개 확보 (1단계) + 뉴스 기반 무한 생성 (3단계)
- 컨셉: **"데이터는 공공 API, 포장은 AI"** → 할루시네이션 리스크 제로
- 정답/팩트는 공공데이터 원본, AI는 오답 보기 생성 + 설명 다듬기만 담당
- 유저 입장에서는 "페이스메이커가 AI로 매일 퀴즈를 내주는" 느낌

## 1단계: 공공 API 데이터 수집 (비용 0, 전부 무료)

### 소스 A. 예탁결제원 금융용어 (~2,000개)

| 항목 | 내용 |
|---|---|
| 엔드포인트 | `http://api.seibro.or.kr/openapi/service/FnTermSvc/getFinancialTermMeaning` |
| 인증 | data.go.kr API 키 (자동승인) |
| 파라미터 | `serviceKey`, `term`(LIKE검색), `pageNo`, `numOfRows` |
| 응답 | `fnceDictNm`(용어명), `ksdFnceDictDescContent`(설명, HTML 포함) |
| 호출제한 | 개발 100회/일, 운영 승격 시 증가 |
| 퀴즈 변환 | "이 용어의 뜻은?" → 정답 1개 + AI가 오답 보기 3개 생성 |

### 소스 B. 한국은행 ECOS 경제용어 (~1,000개)

| 항목 | 내용 |
|---|---|
| 엔드포인트 | `https://ecos.bok.or.kr/api/StatisticWord/{key}/{format}/{lang}/{start}/{end}/{word}` |
| 인증 | ecos.bok.or.kr 회원가입 시 자동 발급 |
| 응답 | `WORD`(용어), `CONTENT`(설명) |
| 호출제한 | 일 10만건 |
| 퀴즈 변환 | 용어-정의 매칭 객관식 |

### 소스 C. 한국은행 경제금융용어 700선 (~700개)

| 항목 | 내용 |
|---|---|
| 형태 | PDF 파일 (무료 다운로드) |
| URL | https://www.bok.or.kr/portal/bbs/B0000249/view.do?nttId=235017&menuNo=200765 |
| 내용 | 700개 용어 + 상세 설명 |
| 퀴즈 변환 | PDF 파싱 → 용어-정의 객관식 |

### 소스 D. 재경부 시사경제퀴즈 (~200개)

| 항목 | 내용 |
|---|---|
| 형태 | CSV 파일 다운로드 (API 아님) |
| URL | https://www.data.go.kr/data/15131287/fileData.do |
| 내용 | OX퀴즈 + 다지선다 혼합, 200문항 |
| 퀴즈 변환 | 이미 퀴즈 형태라 바로 import 가능 |

### 1단계 예상 합계: ~3,900개

## 3단계: 뉴스 기반 AI 퀴즈 자동 생성 (무한)

### 네이버 뉴스 검색 API

| 항목 | 내용 |
|---|---|
| 인증 | 네이버 개발자센터 앱 등록 (Client ID/Secret) |
| 호출제한 | 25,000회/일 |
| 방식 | "금융", "경제", "투자" 키워드로 뉴스 수집 |

### 다음 뉴스 검색 API

- 카카오 개발자센터 앱 등록
- 네이버와 동일 방식

### 자동 생성 플로우

```
크론잡 (매일 아침):
  1. 네이버/다음 뉴스 API → 오늘의 경제 뉴스 수집
  2. 뉴스 본문에서 핵심 키워드/팩트 추출
  3. Claude API로 퀴즈 생성:
     - 정답: 뉴스 속 팩트 (확정)
     - 오답 보기: AI가 그럴듯한 오답 3개 생성
     - 설명: 뉴스 원문 기반 요약
  4. quizzes 테이블에 insert
  5. source = "오늘의 경제뉴스" 태깅
```

→ 매일 자동 추가되므로 퀴즈 고갈 불가

## 퀴즈 배정 로직 (개선)

### 현재 (변경 전)
- 랜덤 셔플 + exclude 방식 → 비효율

### 변경 후: 순번 기반 배정

```
quizzes 테이블에 seq (순번) 컬럼 추가
유저별 last_quiz_seq 저장

새 퀴즈 배정 시:
  → seq > user.last_quiz_seq 인 것 중 7개 (신규)
  → wrong_notes에서 3개 (오답 복습)
  → last_quiz_seq 업데이트

고갈 알림:
  → 어떤 유저든 last_quiz_seq / 전체 퀴즈 수 ≥ 0.8 → 관리자 알림
  → ≥ 0.9 → 긴급 알림
```

## 구현 순서 (다음 세션용)

### Step 1: API 키 발급 (수동)
- [ ] data.go.kr 가입 → 예탁결제원 금융용어 API 키 발급
- [ ] ecos.bok.or.kr 가입 → ECOS API 키 발급
- [ ] 재경부 CSV 파일 다운로드
- [ ] 한은 700선 PDF 다운로드
- [ ] .env에 키 추가: `DATA_GO_KR_API_KEY`, `ECOS_API_KEY`

### Step 2: 수집 스크립트 작성
- [ ] `scripts/fetch-seibro-terms.ts` — 예탁결제원 금융용어 전체 수집 → JSON 저장
- [ ] `scripts/fetch-ecos-words.ts` — ECOS 경제용어 전체 수집 → JSON 저장
- [ ] `scripts/parse-bok700.ts` — 한은 700선 PDF 파싱 → JSON 저장
- [ ] `scripts/parse-quiz-csv.ts` — 재경부 CSV 파싱 → JSON 저장

### Step 3: AI 배치 가공
- [ ] `scripts/generate-quizzes.ts` — 수집된 JSON을 Claude API로 배치 가공
  - 용어 → 객관식 퀴즈 변환 (정답 확정 + 오답 보기 3개 생성 + 설명 다듬기)
  - 카테고리 자동 태깅
  - 결과를 `data/quizzes-batch.json`에 저장

### Step 4: DB 마이그레이션 + Insert
- [ ] `supabase/migrations/004_quiz_seq_and_bulk.sql`
  - seq 컬럼 추가
  - last_quiz_seq 컬럼 추가
  - 기존 30개 퀴즈에 seq 부여
- [ ] `scripts/bulk-insert-quizzes.ts` — 가공된 JSON → quizzes 테이블 bulk insert

### Step 5: 퀴즈 서비스 로직 수정
- [ ] `quiz.service.ts` — getTodayQuizzes 로직 변경 (랜덤→순번 기반)
- [ ] `quiz.service.ts` — last_quiz_seq 업데이트 로직
- [ ] `quiz.service.ts` — 고갈 알림 체크 로직

### Step 6 (3단계): 뉴스 크론잡
- [ ] 네이버/다음 개발자 앱 등록
- [ ] `scripts/daily-news-quiz.ts` — 크론잡 스크립트
- [ ] Render Cron Job 설정
