# 머니런 v2.0 기획서

> **작성일:** 2026-04-11
> **버전:** v2.0
> **상태:** 기획 확정 대기

---

## 1. 프로덕트 개요

**머니런(MoneyRun)** — 비로그인 시뮬레이션 → 로그인 후 AI 개인화 금융 콘텐츠 + 매일 퀴즈/코칭으로 재방문 유도하는 웹 MVP.

### 핵심 변경 (v1 → v2)

| 항목 | v1 | v2 |
|---|---|---|
| 페이스메이커 | AI 메시지 + 퀴즈 10개 + 트랙타일(소비체크/주간요약/월간확정) | AI 메시지(강화) + 퀴즈 1개(출석체크) |
| 마이북 | 상세리포트 + 월간리포트 + 학습콘텐츠 + 스크랩 | 머니레터(상세리포트 + 구매한 책) + 하이라이트 + 스크랩(URL/퀴즈) + 오답노트 |
| 머니북 | 없음 | **신규** — 어드민이 주제별 책 등록, 유저가 구매 시 AI 개인화 |
| 온보딩 | 7개 필드 | 8개 필드 (월 투자액 추가) |
| 퀴즈 | 하루 10개, 난이도 없음 | 하루 1개, 5단계 난이도, 출석체크 연동 |
| 출석/뱃지 | 월간리포트 내 뱃지 | 독립 출석 시스템 + 스트릭 뱃지 |
| 메뉴 | 페이스메이커 / 마이북 / 마이페이지 | 페이스메이커 / 머니북 / 마이북 / 마이페이지 |

### 삭제 기능
- 트랙타일 전체 (매일 소비체크, 주간 요약, 월간 확정/취소)
- 월간 리포트
- 학습 콘텐츠 (learn_contents)
- 학습 스크랩 (user_content_scraps)

### 유지 기능
- 외부 URL 스크랩 (external_scraps) — 기존 유지

### 확장 기능 — 통합 스크랩(북마크)
외부 URL, 구매한 책, 퀴즈 모두 스크랩(북마크) 가능:
- **외부 URL 스크랩**: 기존 기능 유지 (URL → 메타데이터 + AI 요약)
- **책 콘텐츠 하이라이트**: 구매한 책 내 문장별 컬러 스크랩 (책갈피)
- **퀴즈 스크랩**: 흥미로운 퀴즈 문제 북마크 저장

## 3. 온보딩 플로우

### 3-1. 기본 온보딩 (회원가입 직후)

카카오 로그인 → 온보딩 화면 → 8개 필드 입력 → 재무 등급 산출 → 상세 리포트 자동 생성

**입력 필드 (8개):**

| # | 필드 | 타입 | 설명 |
|---|---|---|---|
| 1 | 닉네임 | TEXT | 서비스 내 표시 이름 |
| 2 | 나이 | INT | 만 나이 |
| 3 | 은퇴 예정 나이 | INT | 기본값 55 |
| 4 | 은퇴자금 수령 나이 | INT | 기본값 65 (국민연금 등) |
| 5 | 월 실수령액 | INT(원) | 세후 월급 |
| 6 | 월 투자액 | INT(원) | **신규** — 현재 투자하고 있는 금액 |
| 7 | 월 고정비 | INT(원) | 월세, 보험, 구독 등 |
| 8 | 월 변동비 | INT(원) | 식비, 교통비 등 유동 지출 |

### 3-2. 계산 로직

```
// 사용 가능 예산 (투자액 반영)
availableBudget.monthly = floor1000(실수령액 - 고정비 - 투자액)
availableBudget.daily   = floor1000(monthly / 해당월 일수)
availableBudget.weekly  = floor1000(daily × 7)

// 잉여금
surplus = 실수령액 - 고정비 - 변동비 - 투자액

// 투자 체급 (등급)
expenseRatio = (고정비 + 변동비) / 실수령액
  RED:    expenseRatio ≥ 70%  → 소비케어 집중
  YELLOW: expenseRatio ≥ 50%  → 투자준비
  GREEN:  expenseRatio < 50%  → 투자실습

// 투자 기간 / 수령 기간
investmentPeriod = 은퇴 예정 나이 - 나이
vestingPeriod    = 은퇴자금 수령 나이 - 은퇴 예정 나이
```

### 3-3. 머니북 구매 시 추가 온보딩

머니북에서 책을 구매할 때, 해당 책이 요구하는 추가 데이터를 입력받음.

**예시: "연말정산 완전정복" 구매 시**
- 연 총소득 (근로소득 기준)
- 부양가족 수
- 주거 형태 (자가/전세/월세)
- 신용카드 연 사용액

**예시: "퇴직연금 가이드" 구매 시**
- 현재 퇴직연금 유형 (DB/DC/IRP)
- 퇴직연금 적립금
- 입사일 (근속 연수 계산)

→ 기본 온보딩 8개 필드 + 추가 데이터를 합쳐서 AI가 개인화된 책 내용을 생성.

### 4-2. AI 메시지 생성 로직

**입력 데이터:**
- 유저 재무 프로필 (8개 필드 + 등급)
- 요일별 테마 (월: 예산점검, 화: 절약팁, 수: 소비습관, 목: 금융상식, 금: 주말경고, 토: 자산관리, 일: 회고)
- 등급별 톤 (RED: 직설적/유머, YELLOW: 넛지, GREEN: 응원/도전)
- system_config 상수 (서울 평균 월세, 물가 등)
- 투자액 데이터 반영 (v2 신규)

**금지 표현:** 특정 종목 추천, 확정 수익 보장, "반드시/무조건" 등

**제한:** 하루 2회 (기본 1회 + 새로고침 1회)

### 4-3. 퀴즈 시스템

**하루 1문제, 난이도 5단계**

| 레벨 | 난이도 | 설명 |
|---|---|---|
| 1 | 입문 | 기초 금융 용어, OX 수준 |
| 2 | 초급 | 간단한 금융 개념 |
| 3 | 중급 | 계산이 필요한 문제 |
| 4 | 고급 | 세금/투자 복합 문제 |
| 5 | 전문가 | 실무 수준 금융 지식 |

**퀴즈 플로우:**
```
유저가 페이스메이커 화면 진입
  → 오늘의 퀴즈 1문제 표시 (유저 현재 레벨 기준)
  → 유저 답변 제출

  정답인 경우:
    ✅ "정답입니다!"
    📝 간단 해설 표시
    📌 [스크랩] 버튼 — 이 퀴즈를 북마크
    🔼 "난이도를 올려볼까요?" [올리기] [유지]
    → 출석 체크 완료

  오답인 경우:
    ❌ "아쉽게 틀렸어요!"
    📝 상세 해설 표시
    📋 오답노트에 자동 저장
    📌 [스크랩] 버튼 — 이 퀴즈를 북마크
    🔽 "난이도를 낮춰볼까요?" [낮추기] [유지]
    → 출석 체크 완료 (틀려도 출석 인정)
```

**퀴즈 선택 로직:**
- 30% 확률: 오답노트에서 재출제 (복습)
- 70% 확률: 유저 레벨에 맞는 새 퀴즈
- 이미 푼 퀴즈는 제외

### 4-4. 출석체크 & 뱃지

**출석 규칙:**
- 퀴즈 1문제 풀기 = 당일 출석 완료 (정답/오답 무관)
- 하루 1회만 인정
- 자정(00:00) 기준 날짜 변경

**뱃지 목록:**

| 코드 | 이름 | 조건 | 아이콘 |

| `streak_7` | 일주일 개근 | 7일 연속 출석 | 🔥 |
| `streak_30` | 한 달 개근 | 30일 연속 출석 | ⭐ |
| `streak_180` | 반년 개근 | 180일 연속 출석 | 👑 |
| `total_30` | 출석 30일 | 누적 출석 30일 | 📅 |
| `total_100` | 출석 100일 | 누적 출석 100일 | 🏆 |

## 5. 머니북 (서점)

### 5-1. 개념

어드민이 주제별로 금융 리포트/책을 등록하면, 유저가 목록을 둘러보고 관심 있는 책을 "구매"(현재 무료, 추후 카카오페이 결제 연동)하는 서점 형태.

구매 시 해당 책에 필요한 추가 데이터를 입력받고, 기존 온보딩 데이터와 합쳐서 AI가 **나만의 개인화된 책**을 생성.

### 5-2. 화면 구성

```
┌──────────────────────────────┐
│  🏪 머니북                     │
│                               │
│  [전체] [세금] [퇴직연금]      │
│  [부동산] [주식] [보험]        │
│  ──────────────────────────── │
│                               │
│  ┌────┐ 2026 연말정산          │
│  │ 📗 │ 완전정복               │
│  │    │ 5챕터 · 세금 카테고리  │
│  │    │ "내 상황에 맞는 절세"  │
│  └────┘ [상세보기 →]          │
│                               │
│  ┌────┐ 퇴직연금              │
│  │ 📘 │ 이것만 알면 된다       │
│  │    │ 3챕터 · 퇴직연금      │
│  │    │ "퇴직연금 최적화"     │
│  └────┘ [상세보기 →]          │
│                               │
│  ┌────┐ 월세 VS 전세          │
│  │ 📙 │ 뭐가 유리할까?        │
│  │    │ 4챕터 · 부동산        │
│  │    │ "나에게 맞는 주거"    │
│  └────┘ [상세보기 →]          │
└──────────────────────────────┘
```

### 5-3. 책 상세 & 구매 플로우

```
[1] 유저가 책 상세 페이지 진입
    → 책 제목, 설명, 챕터 목록(미리보기), 구매 여부 표시

[2] "나만의 책 만들기" 버튼 클릭
    → 이 책에 필요한 추가 데이터 입력 폼 표시

[3] 추가 데이터 입력 완료 → 제출
    → 서버: 기본 온보딩 8개 + 추가 데이터 → AI로 챕터별 개인화 콘텐츠 생성
    → "생성 중..." 상태 표시 (비동기)

[4] 생성 완료 → 마이북으로 이동
    → 개인화된 책 열람 가능
```

### 5-4. 책 구조

```
money_book (책 템플릿)
  ├── title: "2026 연말정산 완전정복"
  ├── description: "..."
  ├── category: "tax"
  ├── cover_image_url: "..."
  ├── required_fields: [
  │     { key: "annualIncome", label: "연 총소득", type: "number" },
  │     { key: "dependents", label: "부양가족 수", type: "number" },
  │     { key: "housingType", label: "주거 형태", type: "select", 
  │       options: ["자가","전세","월세"] }
  │   ]
  └── chapters: [
        { order: 1, title: "연말정산이란?", prompt_template: "..." },
        { order: 2, title: "소득공제 전략", prompt_template: "..." },
        { order: 3, title: "세액공제 활용법", prompt_template: "..." },
        { order: 4, title: "놓치기 쉬운 공제", prompt_template: "..." },
        { order: 5, title: "나만의 절세 로드맵", prompt_template: "..." }
      ]
```

각 챕터의 `prompt_template`에는 유저 데이터를 삽입할 플레이스홀더가 포함됨:
```
"{{nickname}}님의 연 소득 {{annualIncome}}원 기준으로, 
 부양가족 {{dependents}}명인 경우의 소득공제 전략을 분석해주세요.
 주거 형태는 {{housingType}}이며..."
```

### 5-5. 카테고리 (초기)

| 카테고리 | 설명 | 예시 책 |
|---|---|---|
| `tax` | 세금/절세 | 연말정산, 종합소득세, 부가세 |
| `retirement` | 퇴직연금 | 퇴직연금 가이드, 연금 최적화 |
| `real_estate` | 부동산 | 월세 vs 전세, 청약 가이드 |
| `stock` | 주식/투자 | ETF 입문, 미국 주식 시작 |
| `insurance` | 보험 | 보험 리모델링, 실손보험 |
| `saving` | 저축/예금 | 적금 비교, 비상금 만들기 |

## 6. 마이북 (내 서재)

### 6-1. 개념

마이북은 유저의 개인화된 금융 라이브러리. "머니레터"라는 이름으로 통합 관리.

**구성:**
- **상세 리포트**: 온보딩 시 자동 생성된 AI 재무 분석 (기존 detailed_reports)
- **구매한 책 목록**: 머니북에서 구매 후 AI 개인화된 책들
- **하이라이트**: 책에서 문장별 컬러 스크랩 (책갈피)
- **스크랩 모아보기**: 외부 URL 스크랩 + 퀴즈 스크랩 통합 보기
- **오답 노트**: 퀴즈 틀린 문제 모음

### 6-2. 화면 구성 — 메인 (머니레터)

```

│  📚 마이북 · 머니레터          │
│  ──────────────────────────── │
│                               │
│  📊 내 상세 리포트             │
│  ┌─────────────────────────┐ │
│  │ YELLOW 등급               │ │
```

### 6-5. 스크랩 → 나만의 책 생성

**스크랩이 100개 쌓이면 "나만의 책" 생성 가능.**

URL 스크랩 + 퀴즈 스크랩 합산 100개 이상이면 버튼 활성화.

```
┌──────────────────────────────┐
│ 📌 스크랩 모아보기             │
│                               │
│ 총 스크랩: 103개              │
│ (URL 67개 + 퀴즈 36개)       │
│                               │
│ ┌─────────────────────────┐  │
│ │ 📖 나만의 스크랩 책 만들기  │  │
│ │                          │  │
│ │ 스크랩한 내용을 바탕으로   │  │
│ │ AI가 나만의 맞춤 금융      │  │
│ │ 가이드북을 만들어 드려요!  │  │
│ │                          │  │
│ │ [나만의 책 만들기] ← 활성  │  │
│ └─────────────────────────┘  │
│                               │
│ ※ 스크랩 100개 미만이면:      │
│ [스크랩 37개 더 모으면 활성화] │
│ ← 비활성 + 진행률 바          │
└──────────────────────────────┘
```

**생성 플로우:**
```
[1] 유저가 "나만의 책 만들기" 클릭
[2] 확인 모달: "스크랩 103개를 바탕으로 나만의 책을 만드시겠습니까?"
    [만들기] [취소]
[3] 서버:
    - 유저의 전체 스크랩 수집 (URL 요약 + 퀴즈 문제/해설)
    - 유저 재무 프로필 (온보딩 8개 필드 + 등급)
    - AI가 스크랩 내용을 분석 → 관심 주제 파악 → 챕터 구성
    - 챕터별 개인화 콘텐츠 생성
    → 비동기 생성 (시간 소요)
[4] 생성 완료 → 마이북 머니레터에 추가
    → 일반 구매한 책과 동일하게 열람 + 하이라이트 가능
```

**AI 챕터 구성 로직:**
```
입력:
  - URL 스크랩들의 aiSummary, channel, title (관심사 파악)
  - 퀴즈 스크랩들의 question, explanation, category (학습 수준 파악)
  - 유저 재무 프로필 (개인화)

AI 프롬프트:
  "이 유저의 스크랩 데이터를 분석해서 관심 주제 Top 3~5를 도출하고,
   각 주제를 챕터로 구성해서 유저 재무 상황에 맞는 맞춤 가이드를 작성하세요.
   유저가 자주 틀리는 퀴즈 카테고리는 더 자세히 설명하세요."

출력:
  - 책 제목 (AI 자동 생성)
  - 3~5개 챕터 (관심 주제별)
  - 각 챕터: 개인화된 분석 + 실행 가이드
```

**규칙:**
- 스크랩 100개 이상일 때만 활성화
- 생성 후 스크랩 카운트는 리셋하지 않음 (계속 누적)
- 여러 번 생성 가능 (스크랩이 추가될수록 새 버전의 책 생성 가능)
- 생성된 책은 "스크랩 기반" 태그로 구분

## 9. 비즈니스 규칙

### 9-1. 퀴즈/출석
- 하루 퀴즈 1문제 한정
- 정답/오답 모두 출석 인정
- 오답 시 자동으로 오답노트에 저장
- 난이도 변경은 유저 선택 (강제 아님)
- 퀴즈 레벨 범위: 1~5 (min 1, max 5)

### 9-2. 머니북
- 현재 무료 구매 (추후 카카오페이 연동)
- 같은 책 중복 구매 불가 (UNIQUE user_id + book_id)
- AI 생성은 비동기 (polling으로 완료 확인)
- 어드민만 책 등록/수정/삭제 가능

### 9-3. AI 생성
- AI는 수치를 만들지 않음 — 유저 데이터 직접 삽입, AI는 분석/서술만
- 금지 표현 필터링 (특정 종목 추천, 확정 수익 보장 등)
- 면책 문구 자동 삽입

### 9-4. 외부 URL 스크랩 — 유튜브 처리

**문제:** 현재 코드는 유튜브 URL을 Claude에 직접 전달하지만, Claude는 영상을 볼 수 없음. URL과 제목만으로 추측 요약하는 상태.

**수정 플로우 (MVP):**
```
유튜브 URL 입력
  → youtube-transcript 패키지로 자막(transcript) 추출
  → 자막 텍스트를 Claude API에 전달
  → Claude가 자막 기반으로 내용 요약 (300자)
  → aiSummary에 저장

  자막 없는 영상:
  → 제목 + 설명(description)만으로 요약
  → aiSummary에 "(자막 없음) 제목 기반 요약" 표시
```

**MVP:** `youtube-transcript` npm 패키지 (무료, API 키 불필요)
- 유튜브 자동생성 자막 지원 (한국어 대부분 가능)
- 수동 자막 우선, 없으면 자동생성 자막 사용

**추후 업그레이드:** YouTube Data API v3
- 공식 API로 자막 + 상세 메타데이터 (조회수, 좋아요, 설명, 태그 등)
- 일일 할당량 제한 있음 (기본 10,000 units/day)
- Google Cloud Console에서 API 키 발급 필요
- 더 안정적이고 정확한 메타데이터 제공
