## 1. 변경 요약

### 삭제된 API (v3.0 → 더 이상 사용하지 않음)
| 삭제된 엔드포인트 | 이유 |
|---|---|
| `POST /course/onboarding/step1` | 카테고리 선택 → 로그인 전으로 이동 |
| `GET /course/onboarding/step2/questions` | → `GET /course/onboarding/quiz/questions`로 변경 |
| `POST /course/onboarding/step2` | → `POST /course/onboarding/quiz/submit`로 변경 |
| `POST /course/onboarding/step3` | 재무 데이터 → 로그인 전으로 이동 |
| `POST /course/onboarding/step4/generate` | → select-level / quiz-submit에 통합 (자동 생성) |
| `GET /course/onboarding/step4/status` | → `GET /course/onboarding/generation-status`로 변경 |
| `POST /course/onboarding/step5/complete` | → `POST /course/onboarding/complete`로 변경 |

### 신규 API
| 엔드포인트 | 설명 |
|---|---|
| `POST /auth/pre-onboarding` | 로그인 전 온보딩 (JWT 불필요) |
| `POST /course/onboarding/select-level` | 기초 시작 / 레벨 찾기 선택 |
| `GET /course/onboarding/quiz/questions` | 진단퀴즈 (힌트 포함) |
| `POST /course/onboarding/quiz/submit` | 퀴즈 제출 + 자동 생성 |
| `GET /course/onboarding/generation-status` | 생성 폴링 |
| `POST /course/onboarding/complete` | 온보딩 완료 |

### 변경된 API
| 엔드포인트 | 변경 내용 |
|---|---|
| `POST /auth/kakao` | `preOnboardingData` 필드 추가 |
| `GET /course/onboarding/status` | 응답 구조 변경 (step1~5 → 플랫 구조) |

### 계산 로직 버그 수정
> **중요:** 고정비에 투자금이 이미 포함됨. 기존에 투자금을 별도 차감하던 버그 수정.
> - 잉여자금 = 수입 - 고정비 - 변동비 (투자금 별도 차감 X)
> - 등급 = (고정비 - 투자금 + 변동비) / 수입 (투자는 지출 아닌 자산 이동)
> - 프론트에서 직접 계산하는 부분 있으면 동일하게 수정 필요

## 2. 전체 플로우

```
[로그인 전 — JWT 없음]
  ① 닉네임 + 관심 분야 선택
  ② 재무 데이터 입력 → 미리보기(현재/미래/행동 탭)
  ③ "무료로 재테크 진단 리포트 발행" 버튼 → 카카오 로그인

[카카오 로그인]
  → preOnboardingData 함께 전송
  → 서버: 재무 프로필 저장 + 상세리포트 자동 생성 + 카테고리 저장

[로그인 후 — JWT 필요]
  ④ "기초부터 시작하기" vs "내 레벨 찾기" 선택 (스킵 불가)
  ⑤ (레벨 찾기 선택 시만) 퀴즈 10문제
  ⑥ AI 마이북 생성 로딩 (자동 시작됨)
  ⑦ 완료 → 페이스메이커 홈
```

## 4. API 명세

### 4-1. 로그인 전 온보딩

---

**`POST /auth/pre-onboarding`** — 리포트 미리보기 (JWT 불필요)

> 프론트에서 수집한 데이터를 서버에 보내면, 계산 결과(미리보기 데이터)를 반환.  
> DB에 저장하지 않음. 프론트가 로컬스토리지에 입력 데이터를 보관.

```typescript
// Request
{
  nickname: string;
  category: '연금' | '주식' | '부동산' | '세금_연말정산' | '소비_저축';
  age: number;
  monthlyIncome: number;        // 월 실수령액
  monthlyInvestment: number;    // 월 투자금
  monthlyFixedCost: number;     // 월 고정비 (투자금 포함!)
  monthlyVariableCost: number;  // 월 변동비
  retirementAge: number;
  pensionStartAge: number;
}

// Response
{
  nickname: string;
  category: string;
  grade: 'RED' | 'YELLOW' | 'GREEN';

  currentTab: {
    grade: string;
    gradeLabel: string;          // '소비케어 집중' | '투자준비' | '투자실습'
    monthlyIncome: number;
    monthlyFixedCost: number;
    monthlyVariableCost: number;
    monthlyInvestment: number;
    surplus: number;             // 잉여자금
    availableBudget: {
      monthly: number;
      weekly: number;
      daily: number;
    };
    fixedCostRatio: number;      // 고정비 비율 (%)
    expenseRatio: number;        // 지출 비율 (%, 투자 제외)
  };

  futureTab: {
    yearsToRetirement: number;
    retirementAge: number;
    pensionStartAge: number;
    pensionGapYears: number;     // 은퇴~연금 수령 사이 공백
    estimatedSavings: Array<{
      label: string;             // '예적금 3%', 'KOSPI 7%', 'S&P500 10%'
      futureAsset: number;       // 예상 총 자산
      monthlyPension: number;    // 예상 월 연금
    }>;
  };

  actionTab: {
    gradeAction: string;         // 등급별 행동 가이드 문구
    ctaMessage: string;          // 가입 유도 메시지
    courseMessage: string;       // 코스 안내 메시지
  };
}
```

**프론트 구현:**
```typescript
// Step 1: 닉네임 + 카테고리 선택 화면
// → 로컬스토리지에 저장
localStorage.setItem('preOnboarding', JSON.stringify({
  nickname, category
}));

// Step 2: 재무 데이터 입력 후
const allData = { ...step1Data, ...step2Data };
localStorage.setItem('preOnboarding', JSON.stringify(allData));

const preview = await POST('/auth/pre-onboarding', allData);

// → 미리보기 화면으로 이동 (현재/미래/행동 탭)
// → "무료로 재테크 진단 리포트 발행" 버튼
```

**브릿지 메시지 (Step 1 → Step 2 사이):**
> 좋은 선택이에요! [category]는 사회초년생이 가장 어려워하지만, 한 번 깨우치면 평생의 자산이 되는 분야죠.  
> 머니런에서는 [category]에 대한 기초, 심화, 마스터 과정을 운영하고 있어요.  
> [nickname]님의 현재와 미래에 대한 재테크 진단 리포트 발행을 위해, 확인이 필요합니다.

**브릿지 메시지 (Step 2 결과):**
> [nickname]님께서 입력한 값까지 고려한, 오직 [nickname]님만을 위한 재테크 진단 리포트가 발행되었어요.

**입력 필드:**

| 순서 | 필드 | 라벨 | 타입 | 비고 |
|------|------|------|------|------|
| Q1 | nickname | 어떻게 불러드릴까요? | 텍스트 | |
| Q2 | category | 금융 공부를 시작하려는 가장 큰 이유 | 선택 (5개) | |
| Q3 | age | 현재 나이 | 정수 | |
| Q4 | monthlyIncome | 월 실수령액(세후) | 원 | |
| Q5 | monthlyInvestment | 월 평균 투자금 | 원 | |
| Q6 | monthlyFixedCost | 투자금을 포함한 월 평균 고정비 | 원 | **투자금 포함** |
| Q7 | monthlyVariableCost | 월 평균 변동비 | 원 | |
| Q8 | retirementAge | 은퇴 예정 나이 | 정수 | |
| Q9 | pensionStartAge | 은퇴자금 수령 예정 나이 | 정수 | |

**카테고리 선택지:**

| 보여줄 텍스트 | 전송 값 |
|---|---|
| 월급 관리가 안 돼요! | `소비_저축` |
| 주식 등으로 자산을 불리고 싶어요 | `주식` |
| 내 집 마련의 꿈을 이루고 싶어요 | `부동산` |
| 세금과 환급금을 똑똑하게 챙기고 싶어요 | `세금_연말정산` |
| 미래의 나를 위해 연금을 준비할래요 | `연금` |

### 4-2. 카카오 로그인 (변경)

---

**`POST /auth/kakao`** — 카카오 로그인 + pre-onboarding 데이터 연동

```typescript
// Request
{
  accessToken: string;
  preOnboardingData?: {          // 로그인 전 온보딩 데이터 (있으면 전송)
    nickname: string;
    category: string;
    age: number;
    monthlyIncome: number;
    monthlyInvestment: number;
    monthlyFixedCost: number;
    monthlyVariableCost: number;
    retirementAge: number;
    pensionStartAge: number;
  };
}

// Response (기존과 동일)
{
  accessToken: string;           // JWT
  user: {
    id: string;
    nickname: string;
    email: string | null;
    isNewUser: boolean;
    hasCompletedOnboarding: boolean;
    onboardingVersion: number;   // 4
  };
}
```

**프론트 구현:**
```typescript
// "무료로 재테크 진단 리포트 발행" 버튼 클릭 → 카카오 로그인
const preData = JSON.parse(localStorage.getItem('preOnboarding') || 'null');

const result = await POST('/auth/kakao', {
  accessToken: kakaoAccessToken,
  preOnboardingData: preData,     // 있으면 같이 보냄
});

// 로그인 성공 후
localStorage.removeItem('preOnboarding');  // 클리어

if (result.user.isNewUser && preData) {
  // → 코스 레벨 선택 화면으로 (/course/level-select)
  router.push('/course/level-select');
} else if (!result.user.hasCompletedOnboarding) {
  // 기존 유저인데 온보딩 미완료 → 상태 조회 후 이어하기
  router.push('/course/level-select');
} else {
  // 기존 유저 → 홈으로
  router.push('/pacemaker');
}
```

**서버에서 일어나는 일 (프론트가 별도 호출할 필요 없음):**
1. ✅ `finance_profiles` 저장
2. ✅ `users` 닉네임 + onboarding_version 업데이트
3. ✅ `onboarding_progress` 생성 (카테고리 저장)
4. ✅ `detailed_reports` 자동 생성 (비동기) → 마이북 머니레터에 상세리포트 표시됨

**`POST /course/onboarding/quiz/submit`** 🔒 — 퀴즈 답변 제출

> 제출하면 레벨 배정 + AI 마이북 생성이 **자동으로** 시작됨.  
> 별도로 생성 API 호출 불필요.

```typescript
// Request
{
  answers: Array<{
    questionId: string;
    answer: number;              // 0-indexed (선택지 인덱스)
  }>;
}

// Response
{
  assignedLevel: '기초' | '심화' | '마스터';
  courseTitle: string;            // "연금 심화 과정"
  correctCount: number;          // 맞힌 개수
  totalCount: number;            // 10
  message: string;               // 레벨별 결과 메시지 (아래 참조)
  wrongNoteMessage: string | null; // 오답 있으면: "오답은 마이북 > 오답노트에 쏙 넣어드렸어요!"
  generationStarted: true;       // AI 마이북 생성 자동 시작됨!
}
```

**레벨 배정 기준 (정답 수):**

| 정답 수 | 레벨 | 메시지 |
|---------|------|--------|
| 0~4개 | 기초 | [닉네임]님은 [카테고리] 기초 단계군요! 훌륭한 기초 지식을 가지고 계시네요. 부족한 2%를 채워줄 '[닉네임]님 전용 [카테고리] 마이북'을 만들어드릴게요. |

## 9. v3.2 퀴즈 스키마 변경 (2026-04-16)

### 변경 요약

| 항목 | Before | After |
|---|---|---|
| 퀴즈 식별 | `id` (UUID)만 사용 | `id` + `quizCode` (`Q00001` 형식) 추가 |
| 난이도 | `difficultyLevel` 1~5 (숫자만) | `difficultyLevel` 1~3 + `difficultyLabel` (초급/심화/마스터) |
| 힌트 | 없음 | `hint` (string \| null) 추가 |
| 통계 | 없음 | `totalAttempts`, `correctCount`, `correctRate` 추가 |

### 영향받는 API 응답

**`GET /quiz/today`** 🔒 — 응답 필드 추가

```typescript
// Response
{
  quiz: {
    id: string;
    quizCode: string;              // ✅ 신규: "Q00001"
    question: string;
    choices: string[];
    hint: string | null;           // ✅ 신규: 힌트
    difficultyLevel: number;       // 1=초급, 2=심화, 3=마스터 (기존 1~5 → 1~3)
    difficultyLabel: string;       // ✅ 신규: "초급" | "심화" | "마스터"
    source: string;
    category: string;
    totalAttempts: number;         // ✅ 신규: 퀴즈 푼 사람 수
    correctCount: number;          // ✅ 신규: 정답자 수
    correctRate: number;           // ✅ 신규: 정답률 (0~100)
  } | null;
  currentLevel: number;            // 1~3 (기존 1~5 → 1~3)
  currentLevelLabel: string;       // ✅ 신규: "초급" | "심화" | "마스터"
  solvedToday: boolean;
}
```

**프론트 대응:**
- `difficultyLevel` 범위가 1~5에서 1~3으로 변경 → 난이도 표시 UI 업데이트 필요
- `difficultyLabel`로 바로 표시 가능 (숫자→텍스트 변환 불필요)
- `hint`는 퀴즈 풀기 화면에 [힌트 보기] 버튼으로 활용
- `totalAttempts`, `correctRate`는 퀴즈 화면 하단에 "N명 참여 · 정답률 65%" 식으로 표시
