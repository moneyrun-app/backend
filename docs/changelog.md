# 기획서 변경 이력 (Changelog)

> backend-plan.md의 버전별 변경 사항을 기록한다.
> 새 기획이 들어올 때마다 여기에 "뭐가 바뀌었는지"를 추가하고, backend-plan.md를 최신으로 갱신한다.

## v1.1 (2026.03.24) — 리뷰 피드백 반영

**변경 배경:**
- 외부 리뷰에서 신호등/소비행동 기준 겹침, 러닝 속도 산식 미정, 개발 순서 부재 지적

**변경된 기능:**
- 신호등 시스템과 커뮤니티 소비행동 그룹을 통일. 별도 판정 없이 신호등 등급 = 커뮤니티 방 배정 (빨강=불난방, 노랑=허리띠방, 초록=투자방)
- 러닝 속도 MVP 임시 산식 확정: (알뜰 블록 일수 / 전체 경과 일수) × 100

**추가된 기능:**
- 개발 순서 (의존성 기반 7단계) 추가

**삭제된 기능:**
- CRON 테이블에서 "커뮤니티 소비행동 그룹 재배정" 항목 삭제 (신호등 재평가에 통합)

**영향 범위:**
- signal/ 모듈이 커뮤니티 방 배정까지 담당. community/ 모듈에서 별도 등급 판정 로직 불필요.

## v1 (2026.03.24) — 초기 기획

최초 작성. 피그마 화면 설계 4장 + 회의록 기반으로 페이지별 백엔드 기능 정리.

**포함된 페이지:**
- 소비블록 페이지 (자산 레이스 시작, 소비 블록 캘린더, 러닝 속도, 거래 내역, 예산 설정)
- 홈 페이지 (AI 페이스메이커 발화, 이번주 성과 뱃지, 자산 형성 속도, 10년 후 예상, 홈 대시보드 통합)
- 커뮤니티 페이지 (소득 3그룹 × 소비행동 3그룹 배정, 피드 CRUD, 태그 자동 추출, Top Creators, 트렌딩)
- 마이북 페이지 (메인배너 롤링, 스크랩 + AI 요약, 검색, 관심 키워드, 추천 콘텐츠)
- 공통 (코드에프 연동, 신호등 시스템, 유저 프로필, AI 모듈, 알림, CRON 작업)

**주요 결정 사항:**
- 신호등 3단계: 빨강(적자) / 노랑(균형) / 초록(잉여)
- 소득 그룹 3개: 기본소득 / 중위소득 / 고소득 (소득세 구간 기준)
- 호스팅: Render
- 기존 자산 제외, 머니런 시작 시점부터의 추가 자산만 추적
- Auth는 프론트가 Supabase SDK로 처리, 백엔드는 JWT 검증만
