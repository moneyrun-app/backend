> **작성일:** 2026-04-15
> **버전:** v3.0
> **상태:** 기획 확정 대기

## 1. 프로덕트 개요

**머니런(MoneyRun)** — 초개인화 온보딩으로 유저의 관심 분야와 수준을 파악한 뒤, 코스 기반 학습(강의·미션·퀴즈·복습) + AI 페이스메이커 코칭으로 재방문 유도하는 금융 학습 웹 MVP.

### 핵심 변경 (v2 → v3)

| 항목 | v2 | v3 |
|---|---|---|
| 온보딩 | 8개 금액 필드 입력 → 끝 | **5단계**: 관심 분야 → 진단퀴즈 10문제 → 데이터 입력 → AI 마이북 생성 → 페이스메이커 연결 |
| 학습 구조 | 없음 (랜덤 퀴즈 + 서점 책) | **코스 시스템**: 카테고리별 기초/심화/마스터 코스 |
| 퀴즈 | 전체 금융 주제에서 난이도별 랜덤 | **코스 카테고리 범위** 내 출제 |
| 페이스메이커 | 변동비/소비 중심 일반 코칭 | **코스 진도 + 미션 수행 기반** 코칭 |
| 마이북 | 서점 구매 책 + 스크랩 | **코스 마이북**(온보딩 생성) + 서점 책 + 스크랩 |
| 머니북(서점) | 유지 | **유지** (코스와 공존) |
| 미션 | 없음 | **신규** — 챕터별 커스텀 미션 |

### 유지 기능
- 머니북(서점): 어드민 등록 책 + AI 개인화 구매
- 외부 URL 스크랩 (유튜브/일반 URL → AI 요약)
- 하이라이트 (5색 문장 스크랩)
- 퀴즈 스크랩/오답노트
- 출석체크 + 뱃지
- 스크랩 100개 → 나만의 책 생성


**레벨 배정 로직:**
```
진단퀴즈 10문제 — 각 문제에 difficulty_weight (1~3)
총점 = Σ(정답 × difficulty_weight)
최대점 = Σ(difficulty_weight)

score_ratio = 총점 / 최대점
  ≤ 30%  → 기초 코스
  ≤ 70%  → 심화 코스
  > 70%  → 마스터 코스
```

### 3-4. Step 3 — 재무 데이터 입력

기존 8개 필드 + 코스별 추가 데이터:

**기본 필드 (8개):**

| # | 필드 | 타입 | 설명 |
|---|---|---|---|
| 1 | 닉네임 | TEXT | 서비스 내 표시 이름 |
| 2 | 나이 | INT | 만 나이 |
| 3 | 은퇴 예정 나이 | INT | 기본값 55 |
| 4 | 은퇴자금 수령 나이 | INT | 기본값 65 |
| 5 | 월 실수령액 | INT(원) | 세후 월급 |
| 6 | 월 투자액 | INT(원) | 현재 투자 금액 |
| 7 | 월 고정비 | INT(원) | 월세, 보험, 구독 등 |
| 8 | 월 변동비 | INT(원) | 식비, 교통비 등 |

**코스별 추가 데이터 예시:**

| 코스 카테고리 | 추가 필드 |
|---|---|
| 연금 | 퇴직연금 유형(DB/DC/IRP), 국민연금 가입 기간, 퇴직연금 적립금 |
| 주식 | 투자 경험(없음/1년 미만/1~3년/3년 이상), 현재 투자 자산, 관심 투자 유형 |
| 부동산 | 주거 형태(자가/전세/월세), 목표 자산(전세/매매), 청약 가점 |
| 세금/연말정산 | 연 총소득, 부양가족 수, 주거 형태, 신용카드 연 사용액 |
| 소비/저축 | 목표 저축액, 비상금 유무, 구독 서비스 수 |

**계산 로직 (기존과 동일):**
```
availableBudget.monthly = floor1000(실수령액 - 고정비 - 투자액)
availableBudget.daily   = floor1000(monthly / 해당월 일수)
availableBudget.weekly  = floor1000(daily × 7)
surplus = 실수령액 - 고정비 - 변동비 - 투자액
grade = expenseRatio 기반 (RED/YELLOW/GREEN)
```

## 4. 코스 시스템

### 4-1. 코스 구조

5카테고리 × 3레벨 = 15개 코스

| 카테고리 | 기초 | 심화 | 마스터 |
|---|---|---|---|
| 연금 | 국민연금 기초 | 연금 절세 전략 | 연금 포트폴리오 최적화 |
| 주식 | 주식 기본 용어 | ETF·배당 투자 | 재무제표·밸류에이션 |
| 부동산 | 전세/월세/청약 | 투자 분석·세금 | 상가/오피스텔 다변화 |
| 세금/연말정산 | 소득세 기초 | 절세·공제 최적화 | 종합소득세·양도세 |
| 소비/저축 | 가계부·비상금 | 소비패턴·자동화 | 재무자유·자산배분 |

### 4-2. 코스 = 마이북 + 미션 + 퀴즈

```
코스 '연금 기초 과정'
  │
  ├── 마이북 (5챕터, AI 개인화)
  │   ├── 1장: 국민연금이란?
  │   ├── 2장: 퇴직연금 기초
  │   ├── 3장: 개인연금 시작하기
  │   ├── 4장: 연금 수령 전략
  │   └── 5장: 나만의 연금 로드맵
  │
  ├── 미션 (챕터별 2~3개)
  │   ├── 1장 미션: "국민연금 예상 수령액 조회하기" (action)
  │   ├── 2장 미션: "내 퇴직연금 유형 확인하기" (action)
  │   ├── 3장 미션: "IRP 계좌 개설 방법 읽기" (read)
  │   └── ...
  │
  └── 데일리 퀴즈 (연금 카테고리에서 매일 1문제)
```

### 4-3. 코스 진행

```
1인 1활성 코스 — 동시 수강 불가

코스 완료 플로우:
  모든 챕터 읽기 + 미션 일정 비율 완료
  → "축하! 연금 기초 과정을 완료했어요!"
  → "다음 코스를 선택해주세요"
     ┌────────────────────────┐
     │ 추천: 연금 심화 과정     │
     │ 또는 다른 분야:         │
     │ [주식 기초] [부동산 기초] │
     └────────────────────────┘
  → 새 코스 시작 → 새 마이북 AI 생성
```

### 5-2. AI 메시지 생성 로직

**입력 데이터:**
- 유저 재무 프로필 (8개 필드 + 등급)
- **활성 코스 정보** (카테고리, 레벨, 현재 챕터, 총 챕터)
- **미션 진행 상황** (전체/완료/현재 챕터 미션)
- 요일별 테마 (월: 예산점검, 화: 절약팁, 수: 소비습관, 목: 금융상식, 금: 주말경고, 토: 자산관리, 일: 회고)
- 등급별 톤 (RED: 직설적/유머, YELLOW: 넛지, GREEN: 응원/도전)
- system_config 상수

**코스 관련 카드 규칙:**
- 5카드 중 최소 1장은 코스 진도/미션 관련
- 미션 미완료 시 넛지: "아직 미션 안 했지? 오늘 해보자"
- 챕터 이동 유도: "3장 다 읽었으면 4장 가볼까?"

**금지 표현:** 특정 종목 추천, 확정 수익 보장, "반드시/무조건" 등

**제한:** 하루 2회 (기본 1회 + 새로고침 1회)

### 5-3. 퀴즈 시스템

**하루 1문제 — 코스 카테고리 범위**

| 레벨 | 난이도 | 설명 |
|---|---|---|
| 1 | 입문 | 기초 금융 용어, OX 수준 |
| 2 | 초급 | 간단한 금융 개념 |
| 3 | 중급 | 계산이 필요한 문제 |
| 4 | 고급 | 세금/투자 복합 문제 |
| 5 | 전문가 | 실무 수준 금융 지식 |

**퀴즈 선택 로직:**
```
1. 유저 활성 코스 확인 → course_category 획득
2. 30% 확률: 오답노트에서 재출제 (course_category 필터)
3. 70% 확률: quizzes에서 course_category + difficulty_level 필터
4. 코스 없는 유저: 기존 전체 카테고리 로직 (하위 호환)
```

### 5-4. 출석체크 & 뱃지

기존 v2와 동일:
- 퀴즈 1문제 풀기 = 당일 출석 (정답/오답 무관)
- 뱃지: streak_7, streak_30, streak_180, total_30, total_100

## 9. DB 스키마 변경

### 9-1. 기존 테이블 수정

| 테이블 | 변경 |
|---|---|
| `quizzes` | `course_category TEXT` 추가 + 기존 category 매핑 |
| `users` | `onboarding_version INT DEFAULT 2` 추가 |
| `pacemaker_messages` | `course_id UUID`, `course_chapter INT`, `mission_context JSONB` 추가 |

### 9-2. 신규 테이블

**`courses`** — 코스 정의 (5카테고리 × 3레벨 = 15개)
```
id              UUID PK
category        TEXT NOT NULL   — '연금'/'주식'/'부동산'/'세금_연말정산'/'소비_저축'
level           TEXT NOT NULL   — '기초'/'심화'/'마스터'
title           TEXT NOT NULL
description     TEXT
chapter_count   INT DEFAULT 5
is_active       BOOLEAN DEFAULT true
created_at      TIMESTAMPTZ DEFAULT now()
UNIQUE(category, level)
```

**`user_courses`** — 유저-코스 연결
```
id              UUID PK
user_id         UUID FK → users
course_id       UUID FK → courses
status          TEXT DEFAULT 'active'   — 'active'/'completed'/'abandoned'
purchase_id     UUID FK → user_purchases
current_chapter INT DEFAULT 1
started_at      TIMESTAMPTZ DEFAULT now()
completed_at    TIMESTAMPTZ
UNIQUE INDEX: (user_id) WHERE status = 'active'   — 1인 1활성 코스
```

**`course_missions`** — 챕터별 미션
```
id              UUID PK
course_id       UUID FK → courses ON DELETE CASCADE
chapter_number  INT NOT NULL
mission_order   INT DEFAULT 1
type            TEXT NOT NULL   — 'action'/'read'/'calculate'
title           TEXT NOT NULL
description     TEXT NOT NULL
created_at      TIMESTAMPTZ DEFAULT now()
UNIQUE(course_id, chapter_number, mission_order)
```

**`user_mission_completions`** — 미션 수행 이력
```
id              UUID PK
user_id         UUID FK → users
user_course_id  UUID FK → user_courses ON DELETE CASCADE
mission_id      UUID FK → course_missions ON DELETE CASCADE
completed_at    TIMESTAMPTZ DEFAULT now()
note            TEXT
UNIQUE(user_id, mission_id)
```

**`onboarding_progress`** — 5단계 온보딩 상태 (이어하기)
```
id                      UUID PK
user_id                 UUID FK → users, UNIQUE
current_step            INT DEFAULT 1           — 1~5
selected_category       TEXT
diagnostic_answers      JSONB
assigned_level          TEXT
course_id               UUID FK → courses
finance_data            JSONB
course_extra_data       JSONB
generation_status       TEXT                     — 'pending'/'generating'/'completed'/'failed'
generation_purchase_id  UUID
pacemaker_welcomed      BOOLEAN DEFAULT false
created_at              TIMESTAMPTZ DEFAULT now()
updated_at              TIMESTAMPTZ DEFAULT now()
```

**`diagnostic_quizzes`** — 진단퀴즈 풀 (카테고리별 10문제)
```
id                UUID PK
category          TEXT NOT NULL
question          TEXT NOT NULL
choices           JSONB NOT NULL
correct_answer    INT NOT NULL   — 1-indexed
difficulty_weight INT DEFAULT 1  — 1~3, 레벨 산출용 가중치
brief_explanation TEXT
created_at        TIMESTAMPTZ DEFAULT now()
```

## 10. 비즈니스 규칙

### 10-1. 코스
- 1인 1활성 코스 — 완료 후 전환
- 코스 콘텐츠는 AI 자동 생성 (추후 어드민+AI 협업)
- 코스 마이북은 user_purchases(source: 'course')에 저장
- 코스와 서점은 독립 — 서점 구매는 코스와 무관

### 10-2. 온보딩
- 5단계 이어하기 보장 (onboarding_progress 테이블)
- 중간 이탈 시 다음 접속에 마지막 단계부터 재개
- v2 레거시 유저(onboarding_version=2)는 기존 기능 정상 작동

### 10-3. 퀴즈/출석
- 하루 퀴즈 1문제 — 코스 카테고리 범위
- 정답/오답 모두 출석 인정
- 오답 시 자동 오답노트 저장
- 퀴즈 레벨 범위: 1~5

### 10-4. 미션
- 챕터별 미션 2~3개 (AI가 마이북 생성 시 자동 생성)
- 미션 유형: action(행동), read(읽기), calculate(계산)
- 완료 여부 페이스메이커가 참조

### 10-5. 머니북(서점)
- 현재 무료 구매 (추후 카카오페이 연동)
- 같은 책 중복 구매 불가
- AI 생성은 비동기 (polling으로 완료 확인)

### 10-6. AI 생성
- AI는 수치를 만들지 않음 — 유저 데이터 직접 삽입, AI는 분석/서술만
- 금지 표현 필터링 (특정 종목 추천, 확정 수익 보장 등)
- 면책 문구 자동 삽입

### 10-7. 일반
- 금액 단위: 원(₩) 정수
- system_config 하드코딩 금지
- ORM 금지 — Supabase 클라이언트 직접 사용
