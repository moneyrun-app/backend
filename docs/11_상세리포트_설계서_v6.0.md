# 상세 리포트 설계서 v6.0

> 온보딩 직후 생성되는 AI 상세 리포트. 3막 스토리라인 + 선물.
> 모든 수치는 서버 계산, AI는 ai_narrative 텍스트만 생성.

---

## 1. 핵심 원칙

1. **AI는 숫자를 만들지 않는다** — system_config + user profile 기반 서버 계산
2. **교육 앱** — 추천 아닌 교육. 확정 투자 권유/특정 상품 지목 금지
3. **면책 문구** 자동 삽입
4. **금액은 원(₩) 정수**, 천원 단위 내림

---

## 2. 스토리라인

```
Step 1. [현재] 나의 '돈 관리' 성적표
  → "너 지금 이래" (현실 직면)

Step 2. [미래] 이대로 가면 만나게 될 나의 모습
  → "이대로면 이렇게 돼" (위기감)

Step 3. [미래] 머니런과 달려갈 새로운 미래 제시
  → "이렇게 바꿀 수 있어" (희망)

선물. 여기까지 읽어주셔서 감사해요
  → 금융 용어 사전 → 마이북에 별도 저장 (보상)
```

| Step | Section | 이름 | 핵심 |

| Step 1 | A | 재무 건강 진단 | 5축 점수 + 또래비교 |
| Step 1 | B | 돈의 흐름 | 워터폴 + 또래비교 |
| Step 2 | C | 통합 시뮬레이션 | 자산성장 + 은퇴 + 생애이벤트 |

## 4. 섹션별 상세 스펙

### Section A. 재무 건강 진단

**목적:** 유저 재무 상태를 5축 점수로 진단 + 또래비교를 자연스럽게 녹임

**입력:**
- user: monthlyIncome, monthlyFixedCost, monthlyVariableCost, surplus, expenseRatio, grade, age, retirementAge
- system_config: peer_avg_expense_ratio, peer_avg_surplus_ratio, peer_avg_variable_ratio (나이대별)

**서버 계산 (5축, 각 20점 만점):**

| 축 | 계산 | 만점 기준 |
|---|---|---|
| 지출 비율 | (1 - expenseRatio) 환산 | 30% 이하 = 20점 |
| 잉여자금 비율 | (surplus / income) 환산 | 30% 이상 = 20점 |
| 고정비 유연성 | 고정비 / 총지출 환산 | 50% 이하 = 20점 |
| 변동비 여유도 | 유저 변동비 비율 vs 또래 평균 | 또래 이하 = 20점 |
| 투자 가용 기간 | (retirementAge - age) 환산 | 25년 이상 = 20점 |

### Section C. 통합 시뮬레이션

**목적:** 자산 성장 + 은퇴 + 생애이벤트를 하나의 타임라인에 통합

**입력:**
- user: surplus, age, retirementAge, pensionStartAge
- system_config: savings_interest_rate, avg_stock_return_kr, avg_stock_return_global, life_event_costs, national_pension_avg_monthly, min_living_cost_retirement, comfortable_living_cost_retirement

**서버 계산:**
```
1) investmentPeriod = retirementAge - age
2) vestingPeriod = pensionStartAge - retirementAge
3) 3시나리오 FV = PMT × ((1 + r/12)^(n×12) - 1) / (r/12)
   - 적금: savings_interest_rate
   - 투자: avg_stock_return_kr
   - 공격: avg_stock_return_global
4) 생애이벤트 합산 = wedding + jeonse + child + car
5) 이벤트 차감 실질 잔액 = FV_at_retirement - totalEvents
6) 공백기 필요자금 = vestingPeriod × 12 × livingCost
7) 공백기 커버 여부 = 실질잔액 ≥ 필요자금
8) 연금 후 월 부족분 = comfortableLiving - nationalPension
```

## 5. 생성 파이프라인

```
[온보딩 완료]
    │
    ▼
[서버: 데이터 수집]
  ├── finance_profiles → 유저 재무 데이터
  ├── system_config → configMap (31개)
  └── getPeerData(age) → 또래 비교 데이터
    │
    ▼
[서버: report-calculator.ts]
  ├── calculateSectionA ~ H → 모든 수치 계산
  └── calculateSectionI → 정적 용어 리스트
    │
    ▼
[서버: report-prompts.ts]
  └── 계산 결과를 프롬프트에 삽입
    │
    ▼
[Claude Sonnet 4.6 — 1회 호출]
  ├── max_tokens: 8000
  └── ai_narrative 9개만 생성
    │
    ▼
[서버: 후처리]
  ├── 계산 결과 + ai_narrative merge
  ├── 금지 표현 필터링
  ├── 면책 문구 삽입
  └── DB 저장 (detailed_reports.sections)
    │
    ▼
[Section I → user_glossaries 테이블 별도 저장]
```

## 6. 비용

| 항목 | 값 |
|---|---|
| AI 호출 | 1회 (ai_narrative만) |

| 리포트 1건 비용 | **~$0.05~0.10** |

## 7. 면책 문구

```
본 리포트는 일반적인 재무 교육 목적으로 제공되며, 특정 금융 상품이나 투자를
추천하는 것이 아닙니다. 모든 투자 결정은 본인의 판단과 책임하에 이루어져야
하며, 필요 시 공인된 재무 상담사와 상담하시기 바랍니다.
```
