# 소비블록 페이지

이 페이지는 머니런의 데이터 뿌리야. 코드에프(MyData)에서 가져온 거래 데이터를 가공해서 유저의 소비 건강 상태를 보여주는 곳.

---

## 자산 레이스 시작

머니런은 기존 자산을 무시하고, 앱 시작 시점부터 새로 만든 자산만 추적해. "다 같이 0에서 시작하는 레이스"라는 컨셉.

**백엔드가 할 일:**
- 유저가 마이데이터를 연동하는 순간, 그 시점의 계좌 잔고 합계를 스냅샷으로 저장해둔다. 이게 Day 0.
- 이후 "현재 총 자산 - 시작 스냅샷"으로 머니런 이후 만든 추가 자산을 계산한다.
- "12일째" 같은 경과 일수도 이 시작일 기준으로 계산.

---

## 소비 블록 캘린더

월간 캘린더에 날마다 빨강(과소비) 또는 파랑(알뜰) 블록이 채워지는 시스템. 이게 머니런의 핵심 시각화.

**백엔드가 할 일:**
- 매일 자정에 전날의 거래 데이터를 집계해서, 그날 지출이 일 예산보다 많았으면 빨강 블록, 적었으면 파랑 블록으로 기록한다.
- 한 주가 끝나면 7일 중 빨강이 많은지 파랑이 많은지로 주간 블록 색상을 확정한다.
- 한 달이 끝나면 주간 블록들을 종합해서 월간 블록 색상을 확정한다.
- 프론트가 요청하면 특정 월의 일별 블록 배열, 주별 요약, 월별 요약을 내려준다.
- "이번 주 목표까지 80,000원 남았어요!" 같은 잔여 예산도 계산해서 내려준다.
- "가소비 7일 / 알뜰 5일" 같은 집계 숫자도 내려준다.

---

## 러닝 속도

소비 블록을 기반으로 만든 지표. 알뜰 블록이 많을수록 러닝 속도가 빨라지는 개념.

**MVP 임시 산식:**
- 러닝 속도(km/h) = (알뜰 블록 일수 / 전체 경과 일수) × 100
- 예: 12일 중 알뜰 5일 → 속도 = (5/12) × 100 ≈ 42 km/h
- 이 산식은 MVP 이후 기획과 함께 고도화할 수 있음

**백엔드가 할 일:**
- 위 산식으로 러닝 속도를 산정한다.
- 같은 소득 그룹 내에서 상위 몇 %인지 순위를 계산한다.
- 전월 대비 속도 증감률도 계산한다 ("+12% this month").
- 진도율 순위는 상위 20%까지만 표기하고, 가장 많은 알뜰 블록을 쌓은 유저에게 고인물 뱃지를 부여한다.

---

## 거래 내역

코드에프에서 동기화한 원본 거래 데이터를 가공해서 보여주는 부분. 투자 내역과 소비 내역이 분리되어 있음.

**백엔드가 할 일:**
- 코드에프 API를 매일 호출해서 신규 거래를 동기화하고 DB에 저장한다.
- 각 거래에 카테고리를 자동으로 분류한다. "배달의민족" → 외식, "스타벅스" → 카페, "월세" → 고정비용, "적금" → 투자 이런 식으로 규칙 기반 매핑. 나중에 AI 분류를 추가할 수도 있음.
- 카테고리와 함께 태그도 자동 부여한다. "선택적 소비", "고정비용", "전략적 자산", "장해물 소비" 같은 태그. 유저가 나중에 수동으로 바꿀 수 있어야 함.
- 투자 내역만 / 소비 내역만 필터링해서 조회할 수 있어야 하고, 월/주차/일별 기간 필터, 카테고리 필터, 페이지네이션을 지원한다.
- 카테고리별 통계도 제공한다 (식비 21%, 카페 6% 같은).

---

## 예산 설정

소비 블록 판정의 기준이 되는 유저별 예산. 온보딩 때 설정하고 나중에 수정 가능.

**백엔드가 할 일:**
- 유저가 월 소득, 고정비, 저축 목표를 입력하면 주간 예산과 일 예산을 자동 계산해서 저장한다.
- 월 예산 = 월 소득 - 고정비 - 저축 목표 → 주간 예산 = 월 예산 ÷ 4 → 일 예산 = 주간 예산 ÷ 7
- 이 예산 값이 소비 블록의 빨강/파랑 판정 기준이 된다.

# 홈 페이지 (페이스메이커 + 러닝트랙)

모든 모듈의 데이터를 종합해서 AI가 유저에게 말을 거는 화면. 머니런의 얼굴.

---

## AI 페이스메이커 발화

화면 상단에 AI 캐릭터가 유저에게 한마디 하는 영역. "오늘 커피 대신 투자하면 10년 후 약 ₩847,000 더 모을 수 있어요!"처럼 구체적인 금액이 들어간 발화.

**백엔드가 할 일:**
- 유저의 현재 상황을 종합해서 AI에게 넘길 컨텍스트를 만든다: 오늘 쓴 돈, 이번 주 블록 현황, 신호등 등급, 예산 잔여액, 학습 현황 등.
- 특정 지출(커피 5,000원)을 감지했으면 그 금액을 10년 복리로 계산해서 "이걸 투자했으면 얼마가 됐을지" 구체 수치를 만든다.
- 그 수치를 목표 달성일 단축 일수로도 환산한다 ("목표달성이 3주 앞당겨져요").
- 이 컨텍스트 + 유저 등급별 톤(빨강이면 따끔하게, 초록이면 격려) 프롬프트를 AI API에 보내서 자연스러운 한마디를 생성한다.
- 발화 내용은 로그로 저장해둔다 (나중에 분석/개선용).

---

## 이번주 성과 뱃지

절약왕, 투자고수, 저축왕인 — 이번 주 유저가 잘한 항목에 뱃지를 부여하는 시스템.

**백엔드가 할 일:**
- 절약왕: 최근 7일간 알뜰 블록이 연속인지 체크.
- 투자고수: 같은 소득 그룹 내 투자 수익률 순위를 계산해서 상위 10%인지 확인.
- 저축왕인: 이번 주 실제 저축액이 목표의 120% 이상인지 확인.
- 매주 월요일에 뱃지를 재계산해서 갱신한다.

---

## 자산 형성 속도 / 목표 달성률

"자산 형성 평균 속도 85 km/h" + "누적 목표 달성률 5%" 카드.

**백엔드가 할 일:**
- 러닝 속도는 소비블록에서 이미 계산한 값을 그대로 가져온다.
- 전월 대비 변화율을 계산한다.
- 목표 달성률은 유저가 설정한 장기 목표 금액 대비 현재 추가 자산의 비율.
- 목표 레이블("경제적 자유")도 함께 내려준다.

---

## 10년 후 예상 자산 / 또래 비교

"10년 후 예상 자산 ₩2.4억, 또래 평균 대비 +34%" 카드.

**백엔드가 할 일:**
- 현재 저축/투자 페이스를 기반으로 복리 계산해서 10년 후 예상 자산을 산출한다.
- 같은 소득 그룹의 평균 자산 형성 속도와 비교해서 "+34%" 같은 비교 수치를 만든다.
- 이 비교를 위해 소득 그룹별 평균 통계를 주기적으로 집계해둬야 한다.

---

## 홈 대시보드 통합

프론트가 홈 화면 진입할 때 한 번에 필요한 데이터를 묶어서 내려주는 통합 API.

**백엔드가 할 일:**
- 위에서 말한 페이스메이커 발화, 이번주 뱃지, 자산 속도, 예상 자산, 소비 블록 요약, 캐릭터 상태를 한 번에 조합해서 응답한다.
- 프론트가 홈에 진입할 때마다 여러 API를 따로 호출하지 않고 이 하나만 호출하면 되도록.

# 커뮤니티 페이지

소득 그룹 × 소비행동 등급으로 나뉜 커뮤니티. 거지방/허리띠방/투자방.

---

## 커뮤니티 배정

유저를 어떤 방에 넣을지 결정하는 로직.

**백엔드가 할 일:**
- 1단계: 소득 그룹 배정 — 유저의 연소득(온보딩 입력 or 코드에프 데이터)을 기준으로 기본소득/중위소득/고소득 3그룹으로 나눈다. 기준은 소득세 과세표준 구간.
- 2단계: 소비행동 그룹은 신호등 등급을 그대로 사용한다. 별도 판정 없음. 빨강 = 불난방, 노랑 = 허리띠방, 초록 = 투자방.
- 마이데이터 연동 전에는 소비성향 TEST 결과로 임시 배정하고, 연동 후 실제 데이터 기반으로 재배정한다.
- 신호등 등급이 바뀌면 커뮤니티 방도 자동으로 따라 바뀐다.
- 소득 그룹 3개 × 신호등 3개 = 총 9개 방이 존재.

---

## 커뮤니티 피드

각 방 안에서 유저들이 글을 쓰고 소통하는 피드.

**백엔드가 할 일:**
- 게시글 CRUD: 작성, 수정, 삭제, 조회. 게시글에는 내용, 태그, 작성시간이 포함.
- 접근 권한 처리: 소속 커뮤니티(같은 소득 + 같은 소비행동)에서만 글 작성/수정/삭제 가능. 비소속 방은 조회/좋아요/댓글/스크랩만 가능. 프론트에 플로팅 버튼 노출 여부를 판단할 수 있도록 권한 정보를 내려준다.
- 좋아요, 댓글, 스크랩(마이북 연동) 기능.
- 태그 자동 추출: 게시글 내용에서 키워드를 뽑아서 태그를 자동 생성한다. 최신순으로 최소 3개~최대 15개.
- 트렌딩 판정: 최근 24시간 내 좋아요/댓글이 급증한 게시글에 트렌딩 표시.
- 익명 프로필: 유저마다 익명 닉네임과 아바타를 자동 생성해서 관리.

---

## Top Creators

활발하게 활동하는 유저를 상위에 노출하는 시스템.

**백엔드가 할 일:**
- 최근 30일간 게시글 수, 받은 좋아요, 받은 댓글을 가중 합산해서 점수를 매긴다.
- 상위 N명을 Top Creators로 선정해서 프론트에 내려준다.
- 매일 재산정.

# 마이북 페이지

유저가 외부 콘텐츠를 스크랩하고, 머니런이 추천하는 교육 콘텐츠도 보는 지식 저장소.

---

## 메인배너

상단 롤링 배너. 유저 스크랩과 머니런 추천 콘텐츠가 번갈아 나옴.

**백엔드가 할 일:**
- 배너 노출 목록을 만들어서 내려준다: 유저 스크랩(A) 1개 + 머니런 추천(B) 1개를 교차 배치.
- (A)는 최근 3개월 이내 스크랩만. 3개월 지난 건 배너에서 빼고 리스트에만 남김.
- (A)가 0건이면 (B)만 노출.
- (B)가 (A)와 URL이 겹치면 중복 체크 후 처리.

---

## 스크랩 기능

유저가 URL을 붙여넣거나 공유하기로 콘텐츠를 저장하는 기능.

**백엔드가 할 일:**
- URL을 받으면 채널을 자동 판별한다 (youtube.com이면 유튜브, threads.net이면 쓰레드, 나머지는 기타).
- URL에서 메타데이터를 추출한다: 제목, 크리에이터명, 썸네일 (유튜브는 YouTube Data API, 나머지는 Open Graph 태그 파싱).
- AI 요약을 비동기로 생성한다. 유튜브는 자막 기반, 쓰레드는 본문 기반, 기타는 HTML 본문 기반.
- 같은 URL을 다른 유저가 이미 요약해놨으면 기존 요약을 재활용한다 (AI 비용 절약).
- 스크랩 저장 시 만료일을 설정한다: 무료 유저는 3개월 후 자동 만료, 유료 구독은 만료 없음.
- 유저가 북마크하면 만료 없이 영구 보존.
- 매일 만료된 스크랩을 soft delete 처리.

### 저장하는 데이터 (기획서 기반):
1. 링크 URL
2. 유입채널 (유튜브/쓰레드/기타)
3. 유입방식 (공유하기/URL복사하기)
4. 크리에이터 계정명
5. 컨텐츠 등록 일자
6. 컨텐츠 제목
7. 유튜브 컨텐츠 내용 요약 텍스트 (최초 공유 시 1회 요약하여 저장)
8. 쓰레드 전체 텍스트

### 주의사항 (기획서 기반):
- 원 출처가 삭제되거나 수정됐을 경우 어떻게 처리할 것인지 확인 필요
- 해당 URL의 중복 스크랩 건수를 추적한다 — 다수의 유저가 스크랩한 콘텐츠를 추천으로 발행하면 위험하므로 별도 집계

---

## 스크랩 목록 조회 + 검색

**백엔드가 할 일:**
- 채널별 필터: All / YouTube / Threads / 커뮤니티 / 기타
- 검색: 제목 + 요약 본문 + 크리에이터 이름으로 검색. 검색 이력이나 추천은 불필요.
- 페이지네이션 지원.

---

## 컨텐츠 상세 보기

스크랩 카드를 클릭하면 AI 요약 텍스트가 확대되고 상세 내용을 볼 수 있음.

**백엔드가 할 일:**
- 해당 스크랩의 요약 텍스트(짧은 버전)와 전체 텍스트(확대 시)를 내려준다.
- "채널 이동" 버튼용 원본 URL을 내려준다.
- 스크랩 삭제 기능.

---

## 관심 키워드

유저가 관심 있는 분야를 설정하는 기능. 청년정책, 투자, 부동산, 대출 등.

**백엔드가 할 일:**
- 유저별 키워드 목록을 저장하고 조회/수정할 수 있게 한다.
- 추후 추천 피드에 이 키워드를 활용할 수 있도록.

## 신호등 시스템 (빨/노/초) = 커뮤니티 소비행동 그룹

앱 전체를 관통하는 유저 등급 시스템. 커뮤니티 배정(불난방/허리띠방/투자방)도 이 등급을 그대로 사용한다. 별도 시스템이 아니라 하나의 판정 결과를 공유.

- 빨강 = 불난방(거지방): 6개월 평균 지출 > 소득 (5% 이상 초과)
- 노랑 = 허리띠방: 6개월 평균 지출 ≈ 소득 (5% 내외)
- 초록 = 투자방: 6개월 평균 지출 < 소득 (5% 이상 절약)

**백엔드가 할 일:**
- 위 기준으로 등급을 판정한다. 이 하나의 결과가 페이스메이커 톤, 커뮤니티 방 배정, 교육 방향에 모두 사용된다.
- 3개월마다 자동 재평가. 적자 전환 시엔 즉시 반영.
- 등급 변화 이력을 기록해둔다.

## 개발 순서 (의존성 기반)

앞 단계가 완성되어야 다음 단계를 만들 수 있는 순서.

```
1단계: auth (JWT 검증) → users (온보딩, 프로필) → 예산 설정
       ※ 모든 기능의 전제 조건

2단계: codef (코드에프 거래 동기화)
       ※ 여기서부터 실제 데이터가 들어옴. 가장 리스크 큰 파트.

3단계: blocks (소비 블록) → signal (신호등 등급 판정)
       ※ 거래 데이터가 있어야 블록 생성 + 등급 판정 가능

4단계: community (커뮤니티 배정 + 피드)
       ※ 신호등 등급이 있어야 방 배정 가능

5단계: pacemaker (AI 페이스메이커) → simulator (복리 계산기)
       ※ 블록 + 신호등 + 거래 데이터를 종합해서 AI가 발화

6단계: mybook (스크랩 + AI 요약)
       ※ 다른 모듈과 독립적이라 5단계와 병렬 가능

7단계: notifications (알림)
       ※ 앱 출시 시점에 맞춰 연동
```
