#### `POST /auth/onboarding` — 월 투자액 필드 추가

```typescript
// Request
{
  nickname: string;
  age: number;
  retirementAge: number;
  pensionStartAge?: number;     // 기본값 65
  monthlyIncome: number;
  monthlyInvestment: number;    // ← 신규 필드
  monthlyFixedCost: number;
  monthlyVariableCost: number;
}

// Response (리포트 생성은 분리됨 — firstReportId 없음)
{
  grade: "RED" | "YELLOW" | "GREEN";
  monthlyExpense: number;
  surplus: number;
  investmentPeriod: number;
  vestingPeriod: number;
  availableBudget: {
    monthly: number;
    weekly: number;
    daily: number;
  };
  // ❌ firstReportId 삭제됨 → 별도 API로 분리
}
```

**중요: 온보딩 → 리포트 생성이 2단계로 분리됨**

기존에는 온보딩 API 하나에서 리포트까지 생성해서 10~15초 걸렸음.
이제 온보딩은 1초 만에 끝나고, 리포트는 별도 요청 → 폴링 방식.

```
온보딩 완료 후 프론트 플로우:

[1] POST /auth/onboarding → 즉시 응답 (1초)
    → 온보딩 완료 화면 or 마이북으로 이동

[2] POST /book/detailed-reports/generate → 즉시 응답
    → 서버에서 AI 리포트 백그라운드 생성 시작
    → 응답: { status: "generating", estimatedSeconds: 15 }

[3] GET /book/detailed-reports/status → 폴링 (3초 간격)
    → { status: "generating" }    ← 로딩 UI 유지
    → { status: "completed", reportId: "uuid" }  ← 완료! 리포트 표시
    → { status: "none" }          ← 생성 요청 안 됨 (에러 케이스)
```

#### `POST /book/detailed-reports/generate` 🔒 — 리포트 생성 요청

```typescript
// Request: body 없음 (유저 재무 프로필 자동 사용)

// Response (202 Accepted)
{
  status: "generating";
  message: string;
  estimatedSeconds: number;    // 약 15초
}

// 이미 생성 중이면
{
  status: "generating";
  message: "리포트 생성 중입니다.";
}

// Error 400: 온보딩 안 됨
{ message: "온보딩을 먼저 완료해주세요." }
```

#### `GET /book/detailed-reports/status` 🔒 — 리포트 생성 상태 확인

```typescript
// Response
{
  status: "generating" | "completed" | "none";
  reportId?: string;       // completed일 때만
  createdAt?: string;      // completed일 때만
}
```

**프론트 구현 예시:**
```typescript
// 온보딩 완료 직후
const onboardingDone = await POST('/auth/onboarding', formData);
// → 마이북 페이지로 이동

// 리포트 생성 요청
await POST('/book/detailed-reports/generate');

// 마이북 페이지에서 폴링
const pollReport = setInterval(async () => {
  const res = await GET('/book/detailed-reports/status');

  if (res.status === 'completed') {
    clearInterval(pollReport);
    // res.reportId로 리포트 카드 표시
    // GET /book/detailed-reports/:reportId 로 내용 조회
  }
  // "generating"이면 로딩 스피너 유지
  // "none"이면 에러 표시 + 재시도 버튼 → POST /generate 다시 호출
}, 3000);
```

#### 퀴즈/출석 API

**`POST /quiz/:id/answer`** 🔒 — 퀴즈 답변 제출 (출석 자동 체크)

```typescript
// Request
{ answer: number; }  // 0~3 (선택지 인덱스)

// Response
{
  correct: boolean;
  correctAnswer: number;           // 정답 인덱스
  briefExplanation: string;        // 간단 해설
  detailedExplanation: string;     // 상세 해설 (오답 시 표시)
  attendanceChecked: boolean;      // 출석 체크 완료 여부
  currentStreak: number;           // 현재 연속 출석일
  suggestLevelChange: "up" | "down" | null;
    // "up": 정답 → "난이도 올려볼까요?"
    // "down": 오답 → "난이도 낮춰볼까요?"
    // null: 변경 제안 없음
}
```

**프론트 구현 포인트:**
- `correct: true` → 정답 축하 + briefExplanation 표시 + "난이도 올려볼까요?" 모달
- `correct: false` → 오답 안내 + detailedExplanation 표시 + "난이도 낮춰볼까요?" 모달
- 유저가 난이도 변경 수락 시 → `PATCH /quiz/level` 호출
- 정답/오답 모두: 📌 [스크랩] 버튼 표시 → `POST /quiz/:id/scrap` 호출

// Response
{
  id: string;
  title: string;
  description: string;
  category: string;
  coverImageUrl: string | null;
  isPurchased: boolean;
  purchaseId: string | null;    // 이미 구매했으면 purchaseId

  chapters: Array<{
    order: number;
    title: string;
    preview: string;           // 미리보기 텍스트 (200자)

→ 유저가 구매하면 `{{변수}}`가 실제 데이터로 치환되고, AI가 개인화된 내용 생성
